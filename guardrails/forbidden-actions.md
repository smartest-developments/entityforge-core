# Forbidden Actions

- Do not overwrite useful repository documentation with generic placeholder text.
- Do not commit generated runtime data, dashboard payloads, or local smoke-test outputs.
- Do not silently change runtime layout, output locations, or audit paths without updating documentation.
- Do not invent commands, frameworks, or setup steps that are not supported by repository files.
- Do not remove resilience or validation logic without an explicit replacement.
- Do not make unrelated edits while handling a scoped task.
- Do not treat historical docs as active source of truth without checking current `core/` code.
